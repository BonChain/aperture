## Decisions

### 2026-06-20 — Source extraction (PRD + Architecture + SRS/domain refs)
- **Product:** Aperture — a disclosure layer for confidential payments on Sui. Hides *amounts*; sender/receiver/timing stay public. Two modes: **Mode A / Auditor Console** (designated read) and **Mode B / Proof-of-Figure** (holder proves a selected aggregate without surrendering their key). *Source: PRD, SRS.*
- **Stakes:** Hackathon PoC (Sui Overflow 2026), institutional/compliance-adjacent context, Devnet only. Not production. *Source: PRD.*
- **Form-factor:** Web app, **desktop-first, no mobile**. React 19 (pinned to Contra's kaisho) + Vite. UI is copy-pruned `kaisho` from Contra; reuses dlog web worker + hooks. State via React Query. *Source: Architecture.*
- **Roles:** Payer/Issuer, Holder/Recipient, Auditor/Verifier. Demo **role-switcher** re-lenses the whole app; lazy wallet signature per role (pre-sign explainer); one wallet drives all three. *Source: PRD, Architecture.*
- **Signature interaction:** same ciphertext renders **open** for Holder vs **closed `•••••`** for Payer/Auditor — visual proof of confidentiality. *Source: Architecture.*
- **Tone guardrail (NFR-9):** "amount-confidential, selectively disclosable" — never "anonymous"/"fully compliant"/"untraceable". Privacy-posture panel enumerates public-on-chain vs auditor-key-only. *Source: PRD, SRS.*
- **Accessibility (FR-7):** no crypto jargon in Holder UI (hide pending/merge/limbs → "funds arriving," "claim," "withdraw"). Dense compliance tables for Auditor. *Source: PRD, SRS.*
- **Mode B gated on SPIKE-1:** proof-of-figure feasibility unproven; UX is specced but flagged. Mode B proof-gen is **client-side**; Mode A decrypt is **server-side**. *Source: Architecture.*
- **Verbatim terminology to mirror:** Org, Recipient, PaymentRun, TokenAccount, EncryptedBalance (active/pending/pendingPublic), AuditLog (audit-trail-of-audit), Disclosure (disclosure receipt), ElGamalNizk, Wrap/Unwrap. *Source: SRS, Architecture.*

### 2026-06-20 — Visual identity (elicited from Tenny)
- **Direction:** Institutional Audit — sober, regulator-grade, quiet authority. Dense tables.
- **Scheme:** **Dark default** (dark institutional canvas; ink/slate with trust-blue accent + verified-green).
- **Role color:** **Distinct accent per role** — Payer = amber, Holder = aqua, Auditor = violet. Role-switch shifts header banner + accent for instant orientation.
- **kaisho base:** **Reskin** — keep kaisho's components/structure, apply fresh Aperture tokens. DESIGN.md owns its own identity; behavioral deltas only in EXPERIENCE.md.
